# Digital-Marketplaace
General Description:

A command-line marketplace, but with digital currency. Users will be able to make an account and log in. Once in the digital marketplace, users can browse listings of items made by other users. If a listing is in their interest, they can choose to buy the listing. Users can also create their own listing. The listing will appear on the browsing screen, and if another user chooses to buy the listing, the listing creator’s currency will increase by the price of the item that was sold. Users can keep track of everything they have done on the digital marketplace through the User Info screen. On the User Info Screen, a user can see how much money they currently have, the listing ID of the items that they have bought, and items that they are currently selling. They can even edit the items that they are currently selling or delete them entirely. 

Goals:

Our goal was to create a program that would simulate a digital marketplace, much like Facebook Marketplace, or the marketplace found in games like Roblox, where Users can buy and sell items. We wanted to create an easy and seamless interface that also kept track of any changes made once a user has quit the program.

Benefits:

The benefit of creating this program is to be able to simulate a system that is widely used in social media and in a vast majority of games. This offers us great practice on how a system like this is created from ideation and how object-oriented programming would play a part in the process. For our users, they are given a place where they can buy and sell their items with ease.


Special Requirements:

All screens are displayed on the command line, and all user input is gathered by using 
Scanners. In order to keep track of User and Listing information and updates, all data in 
The digital marketplace is stored using two txt files, UserData.txt and ListingData.txt. 
After creating an account, buying a listing, creating a listing, or updating anything else on 
the marketplace, the information will be written and changed in one of the two txt files.
